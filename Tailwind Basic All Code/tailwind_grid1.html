Alignment: content, items, self, place-*

Differentiate container vs. item alignment: content-* aligns the full grid; items-* aligns each cell’s contents; self-* overrides individual cells.

justify-content / align-content

Controls free space distribution in the grid container.

justify-items / align-items

Align items inside each grid cell.

justify-self / align-self

Override alignment per item.

place-content / place-items / place-self

Shorthand pairs of justify-* and align-*.

Flex vs Grid (quick mapping)

You listed Flex props too. Here’s a quick mapping so you know when to choose each.

Use Grid when

  • You need two‑dimensional layouts (rows and columns).
  • You want explicit cell placement or template areas.
  • You want equal-height rows by design (auto-rows-fr).

Use Flex when

  • One‑dimensional flow fits (row OR column).
  • You need content-driven wrap and spacing behaviors.
  • You want straightforward alignment along a main axis.

Both support alignment (justify/align), gaps, and ordering. Grid excels at templates and 2D control; Flex excels at simple flows.
